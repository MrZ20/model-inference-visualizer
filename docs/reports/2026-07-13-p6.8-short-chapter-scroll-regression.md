# P6.8 short-chapter scroll continuity regression and repair

> Historical note: P6.9 subsequently extended natural document flow to every chapter. See ADR-0008 and `2026-07-13-p6.9-all-chapter-scroll-regression.md` for the current contract.

## User-visible failure

The Initialize and Tokenize chapters felt stuck while scrolling: the document continued through a long section, but the main scene stayed in the same viewport position. This looked like the page was becoming longer without moving.

## Objective reproduction

The document was scrollable and its height stayed stable at `8601 px`; this was not an overflow lock or asynchronous content growth. The shared chapter layout applied both `min-height: 145vh` and `position: sticky; top: 92px` to every scene. On the shorter Tokenize scene, the document entered a runway where the scene top remained pinned at about `92 px` while scrolling continued.

The page-level CSS regression was written before the repair and failed with 1 failed / 20 passed because no short-chapter override existed.

## Root cause and repair

The general sticky-scrollytelling rule was appropriate for the longer Attention, MoE and Tensor Parallel theaters, but not for the compact Initialize and Tokenize chapters.

Only those two chapters now override the shared rule:

- chapter `min-height` uses its natural content height;
- its `.scene-sticky` canvas uses `position: relative; top: auto`;
- all later chapters retain the shared sticky layout;
- playback, camera takeover, trace data and scene animation are unchanged.

Global smooth scrolling was not changed: it does not control physical wheel scrolling and was not the root cause.

## Real browser evidence

On the rebuilt `http://127.0.0.1:4175/?lang=en` preview at 1280 × 720:

- Initialize: `scrollY 1040 → 1300`, scene top `210.14 → -49.86`, so scroll delta `260` exactly matched visual delta `-260`;
- Initialize transport remained playing and content progress increased `0.064 → 0.131`;
- Tokenize: `scrollY 2140 → 2400`, scene top `210.42 → -49.58`, again `260 / -260`;
- both scenes computed to `position: relative`;
- continued scrolling reached Viewing `Prefill` at `scrollY 3250` while transport remained playing;
- document width stayed equal to the viewport width (`1280 px`).

## Gate

Machine and browser evidence for P6.8 pass. Overall P6 product/visual acceptance remains open until the user personally accepts the updated scroll feel.
