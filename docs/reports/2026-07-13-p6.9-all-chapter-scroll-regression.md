# P6.9 all-chapter scroll continuity regression and repair

## User-visible failure

After P6.8 fixed Initialize and Tokenize, the user observed the same stuck-scroll feeling in other chapters. The remaining scenes still inherited the common `145vh` section runway and sticky canvas.

## Red/green repair

The page regression was promoted from a two-chapter exception to a global contract. Before the repair it failed with 1 failed / 20 passed and printed the live rule `min-height: 145vh`.

The shared rules now use natural document flow:

- `.trace-section { min-height: auto; }`;
- `.scene-sticky { position: relative; top: auto; }`.

The temporary Initialize/Tokenize exception was removed. Focus expansion, playback, camera takeover, scene progress, trace projection and data components were not changed.

## Automated evidence

- targeted page run: 21 tests passed;
- full run: 7 files, 45 tests passed;
- `npm run check`: 0 errors, 0 warnings;
- `npm run build`: adapter-static build succeeded.

## Real browser evidence

On the rebuilt `http://127.0.0.1:4175/?lang=en` preview at 1280 × 720:

- Prefill, Attention, MoE, Tensor Parallel and Decode all computed to `position: relative`;
- Attention: `scrollDelta 180`, `visualDelta -180`;
- Tensor Parallel: `scrollDelta 180`, `visualDelta -180`;
- Decode: `scrollDelta 180`, `visualDelta -180`;
- the document was `7944 px` tall and `1280 px` wide, exactly matching the viewport width;
- all seven chapter scenes remained present with natural heights between about `734 px` and `1120 px` at this viewport.

## Post-repair interaction audit

The final `4175` production preview was reloaded and exercised again after the
global layout change, rather than relying only on the original three-section
sample:

- Initialize, Tokenize, Prefill, Attention, MoE, Tensor Parallel and Decode all
  computed to `position: relative; top: 0px`; every sampled document movement
  produced an equal and opposite scene movement.
- A second pass used real wheel input at each chapter landing point. Initialize,
  Tokenize, Prefill, Attention, MoE and Tensor Parallel each measured
  `scrollDelta 260 / visualDelta -260`; Decode measured `259 / -259` at the
  document-bottom boundary. This proves the global fix with user-like input,
  rather than only programmatic scrolling or a three-chapter sample.
- At `0.5×`, a real wheel gesture moved the document from `scrollY 1027` to
  `1439` while Initialization content progress increased from `0.0118` to
  `0.0197` and the transport remained `Pause inference` (playing). After manual
  takeover, the camera stayed at `scrollY 1451` while content progress continued.
- Browsing to Attention during playback left the transport running and showed
  independent `Viewing: Attention` and `Cursor: Initialize` state.
- Linear Attention, Full Attention, MoE/W8A8 and Tensor Parallel all opened a
  labelled region intersecting the current viewport. Their sequence controls
  changed content-level stage values over time; both TP rank lanes remained
  numerically synchronized.
- The playback options exposed three origins and two modes. Current page + single
  step advanced TP to `1/7`; current step + single step advanced it to `2/7`;
  beginning + continuous reset TP to zero and resumed Initialization.
- English to Chinese preserved the paused Initialization progress exactly at
  `0.80837`, changed the URL to `?lang=zh-CN`, and localized Viewing/Cursor. The
  Chinese Evidence dialog retained modal/focus/Escape behavior.
- Browser warning/error log remained empty; document width still equalled the
  `1280 px` viewport.
