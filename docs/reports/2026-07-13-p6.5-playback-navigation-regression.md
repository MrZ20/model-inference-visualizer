# P6.5 playback origin, page/cursor separation and scroll takeover

> Historical note: the P6.5 “scroll pauses playback” behavior was later superseded by the user's P6.6 requirement. See ADR-0007 and `2026-07-13-p6.6-scroll-without-pausing-regression.md` for the current contract.

## User-visible failure

Browsing from a partially progressed Prefill page to Attention changed Prefill from `0.234` to `1.000`. The single `Play inference` action had no explicit origin or single-step mode, and the top status could not distinguish the page being viewed from the inference step. Wheel/touch listeners existed, but there was no regression matrix proving they still paused playback after decoupling navigation.

The first user sentence ended after “轻微”, so this repair does not invent whether the remaining symptom meant lag, flicker, bounce or another effect. It fixes the reproducible state/scroll race and leaves any more specific residual symptom for user confirmation.

## Root cause

1. `PlaybackSnapshot.chapter` represented both the visible page and the inference cursor.
2. `sceneProgress` treated every chapter before that shared index as 100% complete.
3. Chapter navigation reset progress and could race smooth scrolling, IntersectionObserver and a delayed camera read.
4. Playback start policy was implicit rather than a typed engine command.

## Repair

- Added independent `viewChapter` and inference cursor state plus `progressByChapter`.
- Chapter buttons, Previous/Next and scroll observation now change only the viewed page; deliberate stage selection changes the inference cursor.
- Added three start origins: beginning, current page and current step.
- Added continuous and semantic single-step modes.
- Added an accessible English/Chinese options popover and separate `Viewing` / `Cursor` status.
- Continuous playback still follows the cursor; wheel, touchstart and non-programmatic scroll pause it immediately.

## Red/green evidence

Before the repair:

- page regression: Prefill `0.234 → 1.000` after browsing to Attention;
- three new engine tests failed because `viewChapter`, `progressByChapter` and typed `START` did not exist;
- playback-options page test failed because the control did not exist.

After the repair:

- 7 test files / 41 tests pass;
- `npm run check` reports 0 errors and 0 warnings;
- static production build succeeds.

## Real browser evidence

At 1280 × 720 on `http://127.0.0.1:4175/?lang=en`:

- Prefill stayed `0.234` after browsing to Attention; status changed to `Viewing: Attention` while cursor stayed `Prefill · 23%`.
- Playback-options center hit the real button; the menu was fully inside the viewport (`top 67`, `bottom 344.19`) and exposed five radio controls.
- Current page + single step produced Attention `0.167`, paused.
- After browsing to Tensor Parallel, the cursor still showed Attention `17%`; current step + single step returned to Attention and advanced it to `0.333`.
- During continuous Attention playback, real PageDown scrolling changed transport from playing to paused and left progress stable at about `0.077`.
- English → 中文 kept the selected `beginning` / `continuous` policy and localized all option labels.

## Gate

Machine and browser evidence for P6.5 pass. The user must still confirm whether the previously truncated “轻微…” symptom has any remaining visible form and accept the interaction before P6 is closed.
