# P6.7 Decode light-theme consistency regression and repair

## User-visible failure

The final Decode chapter used a nearly black section, dark scene canvas and dark logits/cache/step cards, while every preceding chapter used a warm light editorial surface. The user correctly reported that the last page did not belong to the same visual system.

## Objective reproduction

The pre-fix production preview reported:

- Prefill/Attention/MoE/TP section backgrounds between `rgb(246–252, 249–252, 247–252)` with dark text;
- Decode section `rgb(23,24,43)` with white text;
- Decode scene `rgb(31,32,55)`.

A page regression read the actual Decode theme rule and failed on `.decode-section { background: #17182b; color: white; }`. The pre-fix targeted result was 1 failed / 19 passed.

## Root cause and repair

Decode had an intentionally isolated dark palette across the section, scene, logits, KV cache and five decode-step cards. It was not caused by inherited variables or a browser rendering issue.

Only the Decode CSS was changed:

- the chapter now uses a faint violet-white surface;
- scene, logits, KV cache and step cards use the existing warm paper and line tokens;
- text uses the shared ink/muted hierarchy;
- active and completed states use the existing indigo/mint tokens;
- the purple selected-token circle remains the primary focal accent.

The trace projection, exact token/logits/KV values, markup and playback logic were not changed.
