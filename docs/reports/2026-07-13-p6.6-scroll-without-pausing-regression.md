# P6.6 scroll without pausing regression and repair

## User-visible failure

The user changed the previous requirement and now requires the long page to remain scrollable while continuous inference keeps running. In the P6.5 build, wheel, touch and non-programmatic scroll explicitly paused transport. In addition, every playback frame rewrote the viewed chapter to the inference cursor, so manual browsing could be pulled back even if native scrolling itself was allowed.

## Red evidence

The regressions were written and run before the repair:

- engine: a manually selected Attention page was overwritten to Initialization on the next playback frame;
- page wheel: Initialization progress stopped after wheel takeover;
- page chapter navigation: clicking Attention during playback changed the transport control back to Play.

The targeted run failed 3 of 31 tests before the implementation changed.

## Repair

- Playback frames now advance only inference cursor state and `progressByChapter`; they do not write `viewChapter`.
- The page owns a separate camera-follow flag.
- Explicit playback start enables camera follow and moves to the chosen origin.
- Wheel, touch, scroll and chapter navigation disable camera follow without sending `PAUSE`.
- While camera follow is disabled, IntersectionObserver updates Viewing independently and the inference cursor continues.
- Explicit Pause and opening a Focus Scene retain their deliberate pause behavior.

## Automated evidence

- targeted engine/page run: 2 files, 31 tests passed;
- full run: 7 files, 43 tests passed;
- `npm run check`: 0 errors, 0 warnings;
- `npm run build`: adapter-static build succeeded.

## Real browser evidence

At 1280 × 720 on the refreshed production preview `http://127.0.0.1:4175/?lang=en`:

- during continuous Initialization playback, real document position changed from `scrollY 1040` to `1760`;
- during the same interval, Initialization content progress increased from `0.087` to `0.138`;
- transport remained `playing`, while Viewing followed the page from Initialize to Tokenize;
- clicking the Attention chapter during playback kept the Pause control visible and the cursor continued from Initialization into Tokenize;
- after manual browsing, `scrollY` stayed at `4228` and Viewing stayed on Attention while Tokenize content progress increased from `0.908` to `0.975`;
- the camera was not pulled back to the inference cursor.

The first browser attempt still showed P6.5 because the pre-existing preview process held the previous hashed entry in memory. The project-owned 4175 preview was restarted after the new static build, then all evidence above was collected in a fresh tab. The unrelated 4173 listener was not changed.

## Gate

Machine and browser evidence for P6.6 pass. P6 product/visual acceptance remains open until the user personally verifies wheel/touch scrolling and accepts the interaction.
